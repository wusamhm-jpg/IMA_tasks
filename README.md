# ima — تطبيق إدارة المهام

مشروع كامل (Full-Stack) لإدارة المهام لصالح شركة **ima**، مبني بـ:

- **الواجهة الأمامية:** HTML + CSS + JavaScript (بدون أي إطار عمل خارجي)
- **الخادم (Backend):** Node.js + Express
- **قاعدة البيانات:** SQLite (عبر وحدة `node:sqlite` المدمجة في Node.js — لا تحتاج تثبيت أي محرك قاعدة بيانات منفصل)

## المزايا
- إدارة كاملة للمهام: إضافة / تعديل / حذف / تحديد كمكتملة
- تصنيفات قابلة للتخصيص بالاسم واللون
- أولويات (منخفضة / متوسطة / عالية)
- تواريخ استحقاق
- تنبيهات تلقائية داخل الواجهة عند اقتراب أو تجاوز موعد الاستحقاق
- فلاتر: الكل / قيد الانتظار / قيد التنفيذ / مكتملة / متأخرة
- بحث فوري في عنوان ووصف المهمة
- لوحة إحصائيات سريعة
- تصميم احترافي بهوية بصرية خاصة بشركة ima (كحل مؤقت حتى تزوّدني بألوان الهوية الرسمية إن وُجدت)

## هيكل المشروع
```
ima-tasks/
├── backend/
│   ├── server.js       # الخادم وواجهات API
│   ├── db.js            # إعداد قاعدة البيانات SQLite والجداول
│   └── package.json
└── public/               # الواجهة الأمامية (يتم تقديمها من نفس الخادم)
    ├── index.html
    ├── style.css
    └── app.js
```

## التشغيل

```bash
cd backend
npm install
npm start
```

ثم افتح المتصفح على:
```
http://localhost:3000
```

قاعدة البيانات (`ima_tasks.db`) تُنشأ تلقائياً عند أول تشغيل، مع 4 تصنيفات افتراضية (عام، عمل، دراسة، شخصي).

> ملاحظة: وحدة `node:sqlite` المستخدمة تتطلب **Node.js 22.5 أو أحدث**. إذا كانت نسختك أقدم، يمكن استبدالها بسهولة بحزمة `better-sqlite3` بنفس بنية الاستعلامات تقريباً.

## واجهات API الرئيسية

| Method | Endpoint              | الوصف                     |
|--------|------------------------|----------------------------|
| GET    | /api/tasks             | جلب المهام (مع فلاتر اختيارية: status, category_id, priority, search) |
| GET    | /api/tasks/stats       | إحصائيات سريعة              |
| POST   | /api/tasks             | إضافة مهمة جديدة           |
| PUT    | /api/tasks/:id         | تعديل مهمة                 |
| DELETE | /api/tasks/:id         | حذف مهمة                   |
| GET    | /api/categories        | جلب التصنيفات               |
| POST   | /api/categories        | إضافة تصنيف                |
| DELETE | /api/categories/:id    | حذف تصنيف                  |
